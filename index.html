Q & A




Web 1.0

  1. The "read" phase
  2. Web pages are static (website doesn't change)
  3. Web pages aren't interactive
  4. Only one way interaction between producer-consumer
  5. Emails were sent through HTML forms

Web 2.0

  1. The "read-write" phase
  2. Web pages are dynamic, responsive to user's input
  3. Web pages are interactive
  4. producer-consumer interaction
  5. APIs were developed for self usage
  6. Supports user-user communication

Web 3.0

  1. The "executable" phase
  2. Web pages are dynamic
  3. Web pages include intelligent and interactive services
  4. Includes AI / Natural Language Proccessing according to the user
  5. Semantics - focus on the meaning of the user's needs and not just the input itself



Examples:
  1. The query "how to set windows" might refer to different content.
    In web 2.0, without semantics (Web 3.0), the results may refer to the operating system - windows.
    This functionality didn't exist in web 1.0

  2. Determining the closest supermarket to me.
    In web 1.0 there wasn't consumer-producer communication, therefore choosing the closest supermarket
    had to be done manually.
    In web 2.0 we could ask for the supermarkets near a ceratin area, and in web 3.0 it is done by
    locating the device used and automatically ordering the results by their distance from my location.

  3. In web 3.0 the results may differ based on the user itself because of the use of semantics.
    In web 2.0 we get the same results for certain query, but in web 3.0 the results differ based
    on the user's properties, for example: age, country, gender, history etc..
